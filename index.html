Steps for running QL catalogs (Docker)

To produce Quick look catalogs, it is necessary to execute three distinct pipelines in series. The procedures for accessing, installing the necessary packages, and running each pipeline are outlined below.

Starting Docker

  1. Step 1: Starting up the docker terminal in an interactive view.
    sudo docker run -it cont_pipe_sm

Pipeline 1

  1. Step 1: Setting up virtual environment

    conda activate myenv

  2. Step 2: Getting ready to run the pipelines The first step is to get a file with the URLs of all VLASS fits images on which the source finding is to be done. One can use get_urls_from_nrao.py for this. The usage is as follows.

    cd $PIPE1 
python get_urls_from_nrao.py <URL> <mode> <imagetype>

    Here, an example for URL is https://archive-new.nrao.edu/vlass/se_continuum_imaging/VLASS2.1 . There are two modes, 'w' for write and 'a' for append. The two options for imagetype are alpha and I. Choose I. The code generates an output file named "manifest.csv". In some cases where there are multiple folders, such as "VLASS2.1" and "VLASS2.2", it may be necessary to run the code a second time in "append mode". This will allow the code to add new entries to the existing "manifest.csv" file instead of overwriting it. Now copy the file, "manifest.csv" to the subfolder media/manifests/.

    cp manifest.csv media/manifests
  3. Step 3: Run the pipeline. Pipeline 1 is written in a modular form, and each of the steps is run in series and in the background. Run the following commands in the specified order:
rm -rf data
python3 catenator.py flush # pipeline cleanup utility
python3 catenator.py configure # This gives you 2 options test and v1. Choose 1.
python pipe1.py #

The logs generated by the program can be found in the data/logs directory. To ensure that each step of the process has been completed successfully, it is recommended to run "python3 catenator.py monitor" after each step. This will allow you to monitor the status of the program and verify that it has finished executing before proceeding to the next step.

Pipeline outputs¶

The final catalogs (with .csv extension) will be present in the folder "data/products". Expected timeline: >~2 weeks

Pipeline 2

  1. Step 1: Run the pipeline. There are five steps and each of the steps is run in series. Run the following commands in the specified order:
    cd ..
. ./pipe2.sh
    Please review the output log files to ensure that the pipeline has not terminated unexpectedly during any of the above steps. It is recommended to examine the output file of each step. As an illustration, you may execute the command tail -300 step1.out after completing the first step.

Pipeline outputs¶

Check out the final outputs of the pipeline 2 in the folder "catalogue_output_files". Expected timeline: 1-2 weeks

Pipeline 3 or SOM pipeline

  1. Step 1: Setting up virtual environment

    conda deactivate
conda activate cutoutenv

  2. Step 2 Copy the output csv file from vlass_cat folder to the current working directory.

    cd $PIPE3_1
cp $PIPE2/catalogue_output_files/*_duplicate_free.csv .

  3. Step 3 Edit cutout_provider_core_old/core/vlass.py to select the relevant epoch by opening in any text editor. Uncomment the relevant line (lines 104 -109). e.g., if ("3.1.ql" in url and "tt0" in url) or ('3.2.ql' in url and 'tt0' in url):

    vim core/vlass.py
  4. Step 4 Downloading the cutouts at source positions using cutout_provider_core. Rename the component catalog name in the file 'splitandcreate.sh' to match the output from previous step.
    vim splitandcreate.sh
. ./splitandcreate.sh
  5. Step 5 Run the pipeline. There are two steps and the first step can be run on the local machine, whereas the second part needs to be run on compute canada.

Before proceeding with the below step that generates the image binary, verify that line 360 (which goes sys.exit(1)) is NOT commented out.

conda deactivate
conda activate sidelobe_pipe_env
cd $PIPE3_2
nohup python3 sidelobe_pipeline.py $PIPE2/catalogue_output_files/*_components.csv components_out_som.csv -p $PIPE3_1/data_out/VLASS/ -s SOM_B3_h10_w10_vlass.bin -n neuron_info.csv -t 10 --cpu 8

Description of the format¶

python3 sidelobe_pipeline.py <catalogue> <outfile> -p <path_to_image_cutouts> -s <som_file> -n <neuron_table> [--cpu] [--overwrite]

where the parameters mean the following

  • catalogue: The name of the input catalogue (csv or fits).
  • outfile: The desired name for the output catalogue.
  • path_to_image_cutouts: The directory that contains all of the VLASS cutouts.
  • som_file: The name of the SOM binary file (SOM_B3_h10_w10_vlass.bin).
  • neuron_file: The name containing the information (neuron_info.csv).
  • --cpu: A Boolean flag. If set, it runs the Mapping stage in CPU mode (Warning: Slow).
  • --overwrite: A Boolean flag. If set, it overwrites the Image and Mapping binaries.

Pipeline outputs¶

  • Image binary (preprocessed images)
  • Catalogue of components that failed preprocessing (fits file)
  • Catalogue of components that passed preprocessing (fits file)
  • Mapping binary
  • Transform binary
  • Final output catalogue

Once the image binaries are created, copy over the *.bin files to Compute Canada, comment out line 360 and run again.

python3 sidelobe_pipeline.py <catalogue> <outfile> -s <som_file> -n <neuron_table> [--cpu] [--overwrite]

Exiting Docker and copying final files

  1. Step 1: Open a new terminal and find out the name of the docker.

    sudo docker ps

  2. Step 2: Replace the 'elastic_khorana' with the name. Change '2SE' to the right epoch and catalog

    sudo docker cp elastic_khorana:/usr/src/app/sidelobe_pipeline/IMG_VLASS2SECIR_components.bin.records.pkl .
sudo docker cp elastic_khorana:/usr/src/app/sidelobe_pipeline/IMG_VLASS2SECIR_components.bin .
sudo docker cp elastic_khorana:/usr/src/app/vlass_cat/catalogue_output_files/VLASS2SECIR_components.csv .
sudo docker cp elastic_khorana:/usr/src/app/vlass_cat/catalogue_output_files/VLASS2SECIR_hosts.csv .
sudo docker cp elastic_khorana:/usr/src/app/vlass_cat/catalogue_output_files/VLASS2SECIR_subtile_info.csv .

Running the final portion of SOM pipeline on Niagra

  1. Step 1: Set up an account on niagra. See 'Getting started' under https://docs.scinet.utoronto.ca/index.php/Niagara_Quickstart . Needs permission from Bryan Gaensler.

  2. Step 2: Copy the component catalog from pipeline 2, and the binary files from pipeline 3 to Niagra.

    Examples are

    • IMG_VLASS2SECIR_components.bin.records.pkl
    • IMG_VLASS2SECIR_components.bin
    • VLASS2SECIR_components.csv
  3. Step 3: ssh into niagra
  4. Step 4: Download and install all necessary software. Replace sebastib with your username
    cd /scratch/b/bmg/sebastib
 git clone https://github.com/CIRADA-Tools/sidelobe_pipeline.git
 export PIPE3_2=/scratch/b/bmg/sebastib/sidelobe_pipeline
 cd $PIPE3_2
 git clone https://github.com/tjgalvin/pyink.git
 cd pyink
 pip install .
 pip install pandas matplotlib reproject scikit-learn gdown
 cd ../..
 gdown https://drive.google.com/uc?id=18ybQF8jOhETmOlQNGbANHf2l_02aLOHR
 unzip transfer.zip
 gdown 'https://drive.google.com/uc?id=15UTRHsLHBK00Q-NZfHbgDOY-SkrSleqa' -O sidelobe.sh
  5. Step 5: Edit sidelobe.sh. Replace instances of sebastib with your username and change your email address
    vim sidelobe.sh
  6. Step 6: Run sidelobe.sh
    . ./sidelobe.sh
